# Diseño: README principal pedagógico

## Audiencia

El lector conoce Python, pero no necesita experiencia previa con MLOps,
contenedores ni Google Cloud.

## Estructura

El documento presenta primero un modelo mental del ciclo de vida. Después explica
cada herramienta en el punto donde resuelve un problema concreto y distingue
código, experimentos, modelos, imágenes y servicios. Las instrucciones operativas
permanecen después de la explicación conceptual.

## Herramientas cubiertas

La explicación incluye Pandas, NumPy, TensorFlow, Keras, Poetry, Optuna, MLflow,
Streamlit, Docker, Git, GitHub, Cloud Storage, Cloud Build, Artifact Registry,
Cloud Run y Google Cloud CLI.
