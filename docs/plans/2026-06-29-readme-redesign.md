# Diseño: documentación operativa del proyecto

## Estructura

El README raíz describe el flujo completo. El README de `parte0/` profundiza en
entrenamiento, Optuna y MLflow. El README de `app/` cubre inferencia, Streamlit y
Docker.

## Plataforma

Los ejemplos priorizan Windows y PowerShell. Cada flujo que depende de Poetry
incluye una alternativa Docker cuando resulta práctico.

## Formato

Cada procedimiento indica requisitos, directorio de ejecución, comandos,
resultado esperado, URL, forma de detenerlo y errores frecuentes. La
documentación no utiliza emojis.
